HTML Tags and Elements

Welcome to HTML ! Get familiar with the Stripe products and explore their features

HTML is a markup language that uses tags, elements, and attributes to describe the structure and layout of a document.

  • Tags are the basic building blocks of an HTML document. They are used to define the structure and layout of the document, and are enclosed in angle brackets (< >). For example, the <p> tag is used to create a paragraph, and the <img> tag is used to insert an image
  • Elements are the individual components of an HTML document, such as headings, paragraphs, and images. Elements are made up of a start tag, content, and an end tag. For example, the following code creates a paragraph element.
  • Attributes provide additional information about an element and are used to modify the default behavior of an element. They are added to the start tag of an element, and are made up of a name and a value. For example, the src attribute is used to specify the source of an image, and the href attribute is used to specify the link of an anchor element.
Basic HTML Document Structure

A basic HTML document has a specific structure that must be followed in order for the document to be properly rendered by a web browser. Here is an example of a basic HTML document structure

                                
 <!DOCTYPE html>
<html>
<head>
<title>My Webpage</title>
</head>
<body>
<h1>Welcome to my webpage</h1>
<p>This is a simple webpage created using HTML.</p>
</body>
</html>
  • The <!DOCTYPE> declaration is used to specify the version of HTML being used. In this case, the <!DOCTYPE html> declaration is used to specify that the document is written in HTML5.
  • The <html> element is the root element of the document and contains all other elements.
  • The <head> element contains information about the document, such as the title of the page, which is displayed in the browser's title bar or tab
  • The <title> element specifies the title of the document, which is displayed in the browser's title bar or ta
  • The <body> element contains the content of the document, such as headings, paragraphs, images, and links
  • The <h1> element is used to create a heading, in this case the title of the webpage.
  • The <p> element is used to create a paragraph.
  • The closing tags are optional for some HTML tags, but it's a good practice to always close them.

This is a basic example and you can add many more tags and attributes to the structure to have a complex webpage, but this is the basic structure that all HTML documents must follow

It's also important to note that HTML5 introduced new semantic tags like <header>, <nav>, <main>, <article>, <aside>, <footer>, among others, that give more meaning to the structure of the webpage, making it more accessible and search engine friendly.

Relative and Absolute URLs with an example

In HTML, URLs can be specified as either relative or absolute. A relative URL specifies the location of a resource in relation to the current page, while an absolute URL specifies the complete location of a resource. A relative URL is specified using a path that starts from the current directory, and it doesn't include the protocol (http/https) or the domain name. Here is an example of a relative URL

                                    
                                <a href="about.html">About Us</a>

In this example, the href attribute is set to about.html, which is a relative URL that specifies the location of the "about us" page in relation to the current page. An absolute URL, on the other hand, includes the protocol and domain name.

Here is an example of an absolute URL

                                     
                                <a href="https://www.example.com/about.html">About Us</a>

In this example, the href attribute is set to https://www.example.com/about.html, which is an absolute URL that specifies the complete location of the "about us" page on the example.com website. It's important to note that relative URLs are more flexible as they allow you to move your website or application from one location to another without changing the URLs, while absolute URLs are more specific and they are useful when linking to external resources.

Another good practice is to add a meaningful text and title attributes to the <a> element, and this will help screen readers to convey the purpose of the link and also it will help the users to understand the context of the link.

It's also recommended to test the links to make sure that they are working correctly and that the URLs are pointing to the correct location.

Linking to Email Addresses with an example

Linking to email addresses in HTML is done using the <a> element and the href attribute, along with the "mailto:" protocol. The "mailto:" protocol is used to create a link that, when clicked, will open the user's email client and pre-populate the recipient's email address and subject line. Here is an example of creating a link to an email address

                                        
                                    <a href="mailto:info@example.com">Contact Us</a>

In this example, the href attribute is set to mailto:info@example.com, which is the email address of the recipient. The text between the opening and closing <a> tags (Contact Us) is the text that will be displayed as the link. You can also add a subject to the email with the subject parameter

                                         
                                    <a href="mailto:info@example.com?subject=Inquiry">Contact Us</a>

You can also add a body to the email with the body parameter

                                        
                                         
                                    <a href="mailto:info@example.com?subject=Inquiry&body=Hello,%0D%0A%0D%0AI%20have%20a%20question%20about%20your%20product.">Contact Us</a>

It's important to note that the mailto: protocol will only work if the user's computer is configured with an email client that can handle the protocol. Another good practice is to add a meaningful text and title attributes to the <a> element, and this will help screen readers to convey the purpose of the link and also it will help the users to understand the context of the link. It's also recommended to test the links to make sure that they are working correctly and that the email addresses are correct
